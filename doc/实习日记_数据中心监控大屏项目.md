# 实习日记 — 数据中心运行监控大屏项目

## 项目背景与目标

本次实习项目是开发一个数据中心运行监控大屏系统。老师提供了四个原始数据文件，分别是服务器主机信息、监控指标字典、性能监控数据和磁盘监控数据，总共将近八万条记录。这些数据格式是制表符分隔的 dat 文件，需要经过一系列处理后，最终以可视化大屏的形式在网页端展示出来，类似于企业中常见的数据中心运维监控界面，能够直观地看到各台服务器的 CPU 使用率、内存占用、网络流量、磁盘 IO 等关键指标的实时状态和历史趋势。

## 思路与实现过程

拿到这个任务后，我首先对四个原始数据文件进行了分析，搞清楚每个文件的字段含义、数据规模以及它们之间的关联关系。基于数据仓库的星型模型思想，我设计了维度表和事实表：主机信息和指标字典作为维度表，性能数据和磁盘数据作为事实表。整个数据处理流程从 Docker 部署 MySQL 开始，通过 LOAD DATA 命令将原始文件导入数据库，然后进行时间戳转换和数据清洗，再利用 SQL 聚合查询生成统计结果。为了避免前端直接查询大数据量的原始表，我额外设计了九张预聚合的结果表，涵盖 KPI 概览、CPU 趋势、内存趋势、网络流量、负载、磁盘 IO、主机排名、机房分布和异常告警等维度，供前端直接调用。

在完成数据层之后，我使用 Node.js 和 Express 框架搭建了后端 API 服务，创建了九个 RESTful 接口，每个接口直接查询对应的 dashboard 结果表，通过 mysql2 连接池管理与数据库的通信。前端方面则采用 Vue3 搭配 Vite 构建工具和 ECharts 图表库，实现了全屏的数据大屏页面。页面布局将排名和分布类面板集中在左侧，五个趋势折线图分布在右侧区域，顶部排列 KPI 数字卡片，底部设置告警滚动条。视觉上采用了深色科技风格，包括半透明玻璃质感面板、霓虹发光边框、动态粒子网络背景等效果，让整个大屏看起来具备数据中心运维场景应有的专业感。

## 技术栈与收获

整个项目涉及的技术栈覆盖了数据处理、后端开发和前端可视化三个层面：Docker 用于部署 MySQL 容器，SQL 用于数据清洗和统计分析，Node.js 加 Express 构建 API 服务，Vue3 加 ECharts 实现前端可视化。通过这个项目，我实践了一套完整的数据处理到可视化展示的全流程，对一些以前只在课堂上学过的概念，比如数据仓库的维度建模、前后端分离的架构设计、Docker 容器的实际使用等，有了更具体的理解和操作经验。尤其是在调试 Docker 文件挂载、MySQL 数据导入、前后端联调等环节中遇到的各种问题，虽然花费了不少时间排查，但也让我对这些工具的底层机制有了更深入的认识。
